제네시스랩
    문제
  • 기존에 제공하는 영상 콘텐츠는 비용 대비 효율이 좋지 않아, 신규 유저 유입과 리텐션을 위한 AI 대화 기능 필요
  • 사용량이 증가할 경우 채팅 응답 지연이(5분 이상) 자주 발생하여 유저의 서비스 경험 하락
  • 스케일 아웃 한계와 블로킹 이슈로 대규모 요청 처리 어려움

    • 해결
  • 1단계: 기존 RDB 기반 Queue 설계의 한계를 해결하기 위해 ThreadPoolTaskExecutor와 CompletableFuture를 사용해 병렬 처리 도입, 응답 속도 개선
  • 2단계: RestTemplate 대신 WebClient를 도입하여 비동기 요청으로 동시처리, 블로킹 이슈를 완화하고 처리량 증가 및 응답 속도 개선
  • 3단계: 기존 구조를 메시지 큐(SQS) 기반 아키텍처로 전환, 채팅 서버와 LLM 간 요청 및 응답을 완전히 비동기화하고 스케일 아웃 문제를 해결

    • 성과
  • 응답 속도를 개선하여 서비스 장애 해결
  • 채팅 서버의 요청 처리량 증가와 시스템 안정성 향상
  • 단계적 개선을 통해 유저 리텐션 및 톡 기능의 사용성 강화로 서비스 품질 향상
  • 레거시한 시스템을 더 많은 트래픽을 감당 가능한 구조로 개선

유모멘트
  • 19만 사용자에게 전송하는데 5시간 걸리던 푸시를 30분 안에 발송할 수 있도록 개선
  • Spring의 @Schedubled를 이용해 1시간마다 등록된 스케줄을 카카오 푸시 API와 FCM, APNs를 활용해 광고 푸시를 전송하고 있었는데, 전체 사용자에게 발송 시 5시간 정도 시간이 소요 되어서, 하루에 1번 이상의 푸시를 발송할 수가 없어서 광고나 이벤트 진행에 문제가 존재하였음
